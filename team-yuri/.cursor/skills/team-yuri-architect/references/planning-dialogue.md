# Architect Planning Dialogue Reference

Use this during PLANNING to transform a project idea into `team-Yuri/plan.md`.

Ask one question at a time. Provide options when useful. Do not assume the user is a software engineer.

Clarify:
- project intent
- primary user
- current state
- constraints
- architecture direction
- phases
- functional testability per phase

Each phase should end with something the user can observe or execute: page, CLI flow, API call, or minimal end-to-end flow.
