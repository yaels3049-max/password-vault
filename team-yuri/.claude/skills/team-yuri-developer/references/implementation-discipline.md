# Developer Implementation Discipline

- Read the architecture contract before coding.
- Read the Manager plan before coding.
- Inspect existing project structure before adding files.
- Follow existing conventions.
- Do not introduce new architecture without approval.
- Do not expand scope.
- Do not leave verification to the user.
- Install dependencies required to run, test, or lint.
- Prefer minimal, focused changes.
- Keep changes traceable to milestones.

Before PASS:
- run relevant unit tests if supported
- run lint if supported
- fix failures within approved scope
- document exact commands and results
- exercise the functional testability path where possible
- update documentation when required
